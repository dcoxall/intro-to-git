Early notes

  • Git is a means of tracking changes to your work
  • GitHub is a place to store those changes away from your machine
  • Tracking changes is fantastic to help you...
    • Undo mistakes
    • Keep multiple versions of your work
    • Remind you of the things you have done

Git Terminology

There are a lot of concepts and taxonomies in git. It helps to get a basic understanding to better grasp how things fit together.

Commits, Branches, Master

  • A commit is a single packaged change
  • A branch is a collection of changes separated
  • Master usualy refers to the main branch

Merging

  • A merge is when 2 branches are brought together
  • A merge usually generates a new commit

Remotes

  • A remote is an external location for git (like GitHub)

Pushing

  • A push is when you publish your changes to a remote

Pulling

  • A pull is when you retrieve changes from a remote to your machine
  • Think of it as checking for updates
